Ockam Command Manual

ockam

ockam [OPTIONS] <COMMAND>

Orchestrate end-to-end encryption, mutual authentication, key management, credential management, and authorization policy enforcement — at scale.

Subcommands:

  • enroll
  • space
  • project
  • status
  • reset
  • node
  • identity
  • tcp-listener
  • tcp-connection
  • tcp-outlet
  • tcp-inlet
  • secure-channel-listener
  • secure-channel
  • forwarder
  • message
  • worker
  • completion
  • vault
  • lease

Options:

  • -h, --help
    Print help information (-h compact, --help extensive)

  • -q, --quiet
    Do not print any trace messages

  • -v, --verbose
    Increase verbosity of trace messages

  • --no-color
    Output without any colors

  • --output <OUTPUT_FORMAT>
    Output format

  • --test-argument-parser

node create

ockam node create [OPTIONS] [NODE_NAME]

Create a node

Arguments:

  • <NODE_NAME>
    Name of the node (Optional)

Options:

  • -f, --foreground
    Run the node in foreground

  • -e, --exit-on-eof
    Watch stdin for EOF

  • -t, --tcp-listener-address <SOCKET_ADDRESS>
    TCP listener address

  • --child-process
    ockam_command started a child process to run this node in foreground

  • --enrollment-token <ENROLLMENT_TOKEN>
    An enrollment token to allow this node to enroll into a project

  • --launch-config <LAUNCH_CONFIG>
    JSON config to setup a foreground node

  • --trusted-identities <TRUSTED_IDENTITIES>

  • --trusted-identities-file <TRUSTED_IDENTITIES_FILE>

  • --reload-from-trusted-identities-file <RELOAD_FROM_TRUSTED_IDENTITIES_FILE>

  • --project <PROJECT>

  • --vault <VAULT>

  • --identity <IDENTITY>

tcp-inlet create

ockam tcp-inlet create [OPTIONS] --at <NODE> --from <SOCKET_ADDRESS> --to <ROUTE>

Create TCP Inlets

Options:

  • --at <NODE>
    Node on which to start the tcp inlet

  • --from <SOCKET_ADDRESS>
    Address on which to accept tcp connections

  • --to <ROUTE>
    Route to a tcp outlet

  • --authorized <AUTHORIZED>
    Authorized identity for secure channel connection

  • --check-credential
    Enable credential authorization. Defaults to the Node's enable-credential-checks value passed upon creation

  • --disable-check-credential
    Disable credential authorization. Defaults to the Node's enable-credential-checks value passed upon creation

  • --alias <ALIAS>
    Assign a name to this inlet

ockam tcp-inlet list [OPTIONS]

Retrieve inlets information on Node

Options:

  • -n, --node <NODE>
    Override the default API node

project addon configure influx-db

ockam project addon configure influx-db [OPTIONS] --endpoint-url <ENDPOINT_URL> --token <INFLUXDB_TOKEN> --org-id <ORGANIZATION_ID>

Configure the InfluxDB Cloud addon for a project

Options:

  • --project <PROJECT_NAME>
    Ockam Project Name

  • -e, --endpoint-url <ENDPOINT_URL>
    Url of the InfluxDB instance

  • -t, --token <INFLUXDB_TOKEN>
    InfluxDB Token with permissions to perform CRUD token operations

  • -o, --org-id <ORGANIZATION_ID>
    InfluxDB Organization ID

  • --permissions <PERMISSIONS_JSON>
    InfluxDB Permissions as a JSON String https://docs.influxdata.com/influxdb/v2.0/api/#operation/PostAuthorizations

  • --permissions-path <PERMISSIONS_JSON_PATH>
    InfluxDB Permissions JSON PATH. Use either this or --permissions

  • --max-ttl <MAX_TTL_SECS>
    Max TTL of Tokens within the Lease Manager [Defaults to 3 Hours]

  • --user-access-role <USER_ACCESS_ROLE>
    Ockam Access Rule for who can use the token lease service

  • --adamin-access-role <ADMIN_ACCESS_ROLE>
    Ockam Access Rule for who can manage the token lease service
